## 1. 目的（広告審査ゲート）

スマートプラス証券（株式会社スマートプラス）への**広告審査提出可否**を独立検証する。所属金融商品仲介業者は r-Laboratory株式会社、所属金融商品取引業者がスマートプラス、加入協会は日本証券業協会という前提。

- GREEN = 「このまま広告審査に提出してよい」
- YELLOW = critical/major 修正必須、minor は記録のみ
- RED = 重大法令違反リスクあり、提出停止すべき

## 3. レビュー対象（広告審査7軸）

### 軸1. 法定表示の充足性 ★最重要★

`index.html` line 1115-1124 footer / `profile.html` line 377-382 footer を起点に：

- 金融商品仲介業者 r-Laboratory の**登録番号**記載があるか（`関東財務局長(金仲)第○号` 形式）
- **所属金融商品取引業者**（スマートプラス）名と**登録番号**記載があるか
- **加入協会名**（日本証券業協会）の記載があるか
- 元本欠損リスク・自己責任原則の文言があるか
- 「投資勧誘を目的としたものではない」旨があるか

### 軸2. 投資勧誘性の判定

LP本文（特に `index.html` line 917-965 の Step 1 比較表周辺）を以下で判定：

- 「Aが有利」「繰上げ＋運用が有利」等の**断定的優位性表示**の有無
- 特定の運用商品・運用手法を推奨しているか（新NISA = 商品カテゴリーであり、推奨の射程に入りうる）
- 「年利6%想定」（line 958）が将来利回り保証と誤認されるか
- **比較表（line 931-955）が「投資勧誘」に該当する可能性**を3段階で判定（強／中／弱）

### 軸3. 試算条件・前提の開示

`index.html` line 931-955 の比較表について：

- 試算前提（仙台市・令和7年度料率、年利6%、家族構成、他収入、医療費）の**完全性**
- 「あくまで一例」「個人差あり」「将来利回り無保証」の注記が**比較表直下**にあるか
- NISA 月積立額（11.4万円 or 別額）の明示有無
- 出典明記（厚労省モデル年金額、国保料率等）

### 軸4. 景表法「無料」表記

`index.html` line 877, 895, 1066, 1085 / `profile.html` line 369 の「無料」表記を全件抽出：

- 「無料」のすぐ近くに「2回目以降は有料」等の条件併記があるか
- 「初回のみ無料」が hero・header の near distance で読み取れるか

### 軸5. 誇大・優位性誇示・他業界比較

- `index.html` line 1026 「税理士からも、社労士からも、金融機関の窓口からも、なかなか出てこない」 → 他士業・他業界を否定的に言及していないか
- `profile.html` line 323 「30年以上、お金の出口を一緒に考える仕事」 → 経歴と整合するか（1990年安田火災入社・2013年LSFP設立）
- 資格・称号の誇大表現

### 軸6. 固有概念表記のリスク

- `index.html` line 1026 **「老娯資金最大化プランナー」** ：加藤さん固有の意図的造語。スマートプラス審査員には「老後」の誤字に見える可能性。**広告審査の差し戻しリスクを 強/中/弱 で判定**。
- 「ライフスケッチ®」（`profile.html` line 350）：® 表記の整合性確認

### 軸7. 個人情報・問い合わせ・予約導線

- `index.html` line 1098 Google Calendar 予約リンクのみ。**個人情報の取扱い方針**（プライバシーポリシー）リンクが必要か（Google Calendar 経由で氏名・メール等を取得するため）
- 予約完了後の連絡フローが明示されているか

## 4. 検証手順

1. 軸1（法定表示）から開始。**該当行を grep で物理的に特定**してから判定。
2. 軸2（投資勧誘性）は比較表周辺を**強・中・弱**で 3 段階判定。法令解釈に揺れがある領域は「グレー」と明記してOK。
3. 軸4（無料表記）は全件 grep 抽出してから条件併記の有無を判定。
4. 既存 `CODEX_REVIEW_BRIEF.md` の指摘と**重複する場合は「(重複) 既存briefで指摘済」と明記**してスキップ。
